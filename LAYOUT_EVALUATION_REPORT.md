# Survey Agent Simulator 布局评价报告

评审日期：2026-06-20  
评审环境：macOS（Apple Silicon）、Next.js 16.1.6、本地开发服务器、应用内浏览器  
测试视口：桌面端 1280 × 720；移动端 390 × 844

## 结论摘要

项目的桌面端工作台思路成立，问卷模式用“配置—结果—分析”三栏形成清晰的工作流，访谈模式进一步拆成“配置—受访者—对话—分析”四个任务区。暗色数据驾驶舱风格统一，青色主色能有效指示当前动作与关键状态。

当前最明显的问题是布局只为宽屏桌面设计，没有可用的响应式降级。390 px 视口下页面实际宽度达到 881 px，产生约 491 px 的横向溢出，用户只能横向拖动查看后续栏位。综合评价：**桌面端 7.5/10，移动端 3/10，整体 6.5/10**。

## 实测结果

### 桌面端（1280 × 720）

- 问卷模式采用 340 px 左栏、弹性中栏、340 px 右栏；首屏能同时看见配置、问卷总览和分析面板。
- 访谈模式的中间区域再拆分为受访者列表和对话详情，四区边界明确。
- 页面无横向溢出：文档宽度与视口宽度均为 1280 px。
- 页面实际滚动高度约 1260 px，超过 720 px 视口。主内容设置了视口高度，但子面板继续向下撑出文档，首屏无法看到完整受访者配置和底部主操作。
- 两种模式切换正常；浏览器控制台未发现 error 或 warning。

### 移动端（390 × 844）

- 文档实际宽度为 881 px，超过视口 491 px，存在严重横向溢出。
- 三栏/四栏结构没有折叠、堆叠或抽屉化，右侧结果和分析区被留在屏幕外。
- 顶部品牌、模式切换和状态胶囊被挤在同一行，标题换行，模式文字几乎逐字竖排，信息可读性显著下降。
- 左侧配置区仍保持接近桌面宽度；它本身可操作，但不能代表整个工作流在移动端可用。

## 做得好的地方

1. **任务结构直观。** 左侧负责输入，中间负责过程/结果，右侧负责分析，符合研究工作流的自然顺序。
2. **模式差异表达清楚。** 问卷模式强调批量统计；访谈模式增加受访者列表和对话区，信息架构能反映业务差异。
3. **视觉语言一致。** 暗色底、低饱和边框、青绿色强调色、圆角卡片和统一图标形成稳定的产品感。
4. **空状态覆盖完整。** 样本、结果、问卷详情、图表等区域在无数据时都有说明，不会出现意义不明的空白。
5. **主操作醒目。** 左栏底部的“开始问卷模拟/开始访谈模拟”使用高亮色，动作优先级明确。

## 主要问题与优先级

### P0：缺少响应式布局

`app/page.tsx` 的主区域始终使用横向 `flex`，左右栏固定为 340 px，没有任何断点规则。移动端不仅视觉拥挤，而且结果区和分析区默认不可见。

建议：

- 小于 768 px：改为单栏；配置、运行结果、分析通过顶部 Tabs 或底部导航切换。
- 768–1199 px：保留主工作区，把配置和分析移入可开合 Sheet/Drawer。
- 1200 px 以上：保留现有三栏/四栏桌面工作台。
- 为主容器和各栏补充 `min-w-0`、`overflow-hidden`，避免内容反向撑宽父级。

### P1：垂直滚动模型不统一

主区域声明为 `h-[calc(100vh-73px)]`，但子内容仍把文档撑到约 1260 px。用户向下滚动后会失去顶部模式与状态上下文，左栏主操作也需要滚到页面底部才能到达。

建议：桌面端让页面骨架固定在视口内，各栏分别使用 `h-full min-h-0 overflow-y-auto`；左栏主操作使用栏内 sticky footer。若选择整页滚动，则移除固定视口高度，避免两种滚动模型混用。

### P1：桌面端中间工作区在较窄屏幕上偏紧

1280 px 下扣除两侧 680 px 后，中栏约 600 px。访谈模式还会把这部分继续拆成列表和对话区，空状态尚可，但真实长文本对话会更拥挤。

建议：为桌面布局设置合理的最小宽度（建议 1200–1280 px），在更窄的桌面/平板宽度自动收起右侧分析栏；访谈列表可允许折叠为头像/编号窄栏。

### P2：顶部栏在窄宽度下没有重排策略

品牌区、模式切换和运行状态固定同排。移动端会导致标题换行、控制项被压成极窄胶囊。

建议：移动端将品牌和状态放第一行，模式切换放第二行并占满宽度；副标题可隐藏或缩短。

### P2：小字号与低对比文本偏多

界面大量使用 10–11 px 的说明、按钮和状态文字，在暗色背景下阅读负担较高，尤其是统计卡说明和空状态。

建议：正文/控件文字尽量不低于 12–14 px；仅把非关键元数据降到 11–12 px，同时提高 muted 文本与背景的对比度。

### 已处理：部署依赖的跨平台问题

初次部署时，`package.json` 直接声明了 Windows 专用的 `@next/swc-win32-x64-msvc`，导致当前 macOS 环境执行普通 `npm install` 报 `EBADPLATFORM`，强制安装后也出现不完整的 `next` 包。

本次已删除该平台专用的直接依赖并更新锁文件。普通 `npm install` 随后成功，开发服务器稳定返回 HTTP 200。后续仍建议在 macOS/Linux CI 上用干净环境验证 `npm ci && npm run build`，防止平台依赖回归。

## 推荐改版顺序

1. 先修 P0 响应式骨架，确保移动端不横向溢出。
2. 统一桌面端滚动模型，并固定关键操作和状态上下文。
3. 为 768–1199 px 增加“中心工作区 + 可开合侧栏”的平板/小桌面形态。
4. 最后调整字号、对比度、标题截断和空状态密度。

## 验收标准

- 390、768、1024、1280、1440 px 宽度下均无非预期横向滚动。
- 390 px 下可在不横向拖动的情况下完成：配置问卷、启动模拟、查看结果、查看分析。
- 桌面端任一栏滚动时，模式、运行状态和主操作仍易于找到。
- 真实长标题、长问题和长对话不会遮挡按钮或撑破栏宽。
- 干净 macOS/Linux 环境可直接执行 `npm ci` 和 `npm run build`。
