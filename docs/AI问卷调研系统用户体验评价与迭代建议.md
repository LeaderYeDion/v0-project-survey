# AI 问卷调研系统用户体验评价与迭代建议

## 1. 评估说明

### 1.1 评估视角

本文只从目标用户视角评价产品，不评价 mock 回答、情绪判断或统计结果本身是否真实、准确，也不讨论历史代码与技术实现。

假设用户是一名研究公共政策的高校博士或教授，准备研究一项公共政策对公众态度和行为的影响。由于真实样本招募慢、成本高，公开数据存在滞后，希望先用 AI 模拟问卷或访谈完成探索性研究，以便：

- 发现问卷或访谈提纲中的问题；
- 预判不同群体可能出现的态度差异；
- 形成待验证的研究假设；
- 决定后续真实调研应重点覆盖哪些人群和问题；
- 生成可保存、可比较、可导出的研究过程材料。

### 1.2 实际体验范围

本次评估在浏览器中实际完成了以下操作：

1. 进入系统并选择问卷模式；
2. 将调研标题和说明改为“城市拥堵费政策公众态度调研”；
3. 查看表单式问题配置、受访者配置和 JSON 配置；
4. 启动 7 位虚拟受访者的问卷模拟，观察运行中状态与实时结果变化；
5. 查看单份问卷、问题分析、性别筛选和按性别分组；
6. 触发保存、历史记录和 JSON 导出入口；
7. 切换到访谈模式，浏览受访者列表和一位受访者的完整对话；
8. 查看概览、问题分析和人群分析。

因此，以下判断来自真实交互，而不是从产品说明或代码推断。

## 2. 总体评价

当前系统已经具备一套可演示的“配置—运行—查看个体—查看汇总—筛选分析—保存/导出”闭环，尤其是问卷与访谈两种结果视图、实时运行反馈、人口维度筛选，能让用户迅速理解产品想做什么。

但从严肃研究用户的角度看，它目前更像一个“AI 调研过程与结果的可视化演示台”，还不是一个可以放心用于探索性研究的“研究工作台”。主要问题不在于功能数量少，而在于研究主线缺失：系统没有在运行前帮助用户确认研究设计是否完整，也没有在运行后清楚回答“本轮模拟覆盖了谁、哪些结果可比较、为什么有人终止、与上一轮相比发生了什么、下一步应该做什么”。

当前最需要的不是继续增加图表和装饰性指标，而是把产品重心从“展示模拟正在发生”转向“帮助研究者设计、审查、比较和沉淀一轮模拟研究”。

若以目标用户的核心任务完成度评分：

| 核心任务 | 当前评价 | 说明 |
| --- | --- | --- |
| 快速搭建一轮模拟调研 | 3/5 | 可编辑问题和人群，也支持 JSON，但缺少政策研究模板、导入和设计检查 |
| 理解模拟覆盖了哪些人 | 3/5 | 有人口配置和人群分析，但样本结构、配额及交叉覆盖不够直观 |
| 观察模拟是否顺利运行 | 4/5 | 状态、进度、完成/终止数量反馈及时 |
| 从结果中形成研究判断 | 2/5 | 展示多，但全局、筛选和分组之间层级不够清晰，缺少结论工作台 |
| 审查个体回答和异常 | 3/5 | 可看单份问卷和完整访谈，但查找、标注、筛选异常较弱 |
| 保存、复现和比较研究 | 1/5 | 保存/历史/导出入口存在，但缺少清晰反馈、版本和对比能力 |
| 为下一轮真实调研做准备 | 1/5 | 缺少问卷诊断、样本缺口、假设清单和下一轮建议 |

## 3. 对六个核心问题的回答

### 3.1 使用过程中，我最关注什么

### 第一，模拟对象是否与研究对象一致

我首先关注的不是回答内容，而是本轮样本到底代表谁：地区、年龄、职业、收入、政策利益相关方、是否直接受政策影响、是否了解政策、城乡/户籍、教育程度等维度如何分布，各群体各有多少人，是否存在明显遗漏或配额失衡。

当前系统允许配置性别、年龄、职业、城市、收入和人数，这是一个起点，但公共政策研究往往更依赖“与政策的关系”，例如拥堵费研究需要区分通勤方式、居住区、工作区、驾车频率、是否有替代交通、政策成本承担者等。固定人口字段不能充分表达研究对象。

### 第二，研究设计是否可执行、可解释

我需要在运行前确认：题目类型是否合适、选项是否完整或互斥、量表方向是否一致、是否有引导性表述、题目顺序是否可能产生偏差、访谈是否允许追问、终止规则是什么。

当前配置界面能编辑题目，却没有“研究设计审查”这一层。用户可以顺利按下开始，但并不知道自己启动的是不是一轮结构合理的调研。

### 第三，运行过程是否受控

我关心当前完成了多少、哪些人正在回答、为什么终止、能否暂停、能否取消、能否只重跑失败样本、预计还需多久、运行中的配置是否被冻结。

系统提供了实时进度、完成数、终止数和当前状态，视觉反馈较清楚；但运行控制与异常处理不足。出现终止时只能看到数量，不能立即理解原因与影响，也不能对失败样本采取动作。

### 第四，结果是否能支持比较和决策

我关心的不是“有多少张图”，而是：总体态度是什么，不同群体差异在哪里，差异基于多少样本，有哪些值得真实调研验证的假设，哪些发现可能只是样本配置造成的。

当前系统能筛选人口维度并进行分组，这一方向是正确的。但“回答分布”“筛选洞察”“分组结果”同时出现，信息层级不够明确。实际筛选性别后，全局回答数仍保留，同时另外出现筛选洞察；研究者需要自行判断哪个数字对应当前筛选，容易误读。

### 第五，研究过程能否复现和追溯

我需要知道这轮调研使用了哪一版问卷、哪一版人群配置、何时运行、使用何种模拟设置、与上一轮有何差异，并能完整导出原始记录和汇总结果。

当前虽有 JSON/CSV、保存和历史入口，但体验上缺乏强确认：点击保存后没有形成清楚的“已保存版本”；打开历史记录时，弹窗内也没有清楚呈现刚保存的记录。点击导出后也缺少可见的成功、文件名、导出范围或失败反馈。这会直接削弱研究信任。

### 3.2 我期望系统提供什么信息、帮我完成什么工作

我期望系统围绕一轮研究的生命周期提供以下能力：

### 研究设计阶段

- 从研究目标生成或导入问卷/访谈提纲；
- 支持政策研究所需的自定义受访者属性和配额；
- 展示样本结构预览与覆盖缺口；
- 检查重复题、引导性问题、量表不一致、选项遗漏、逻辑冲突；
- 明确问卷模式与访谈模式各自适用的场景和差异；
- 允许保存为草稿、复制上一轮、创建版本。

### 运行阶段

- 显示总任务、完成、终止、失败、预计剩余时间；
- 允许暂停、继续、取消和重试失败样本；
- 展示终止/失败原因及其对样本结构的影响；
- 运行时锁定配置，避免结果与配置版本错位；
- 允许先用少量样本试跑，再扩展到正式模拟。

### 分析阶段

- 默认给出一个清晰的研究摘要，而不是让用户自己拼接多个面板；
- 显示总体态度、关键分歧群体、异常回答、未回答与终止情况；
- 支持任意自定义维度的筛选、交叉分组和样本数提示；
- 明确区分“总体结果”“当前筛选结果”和“组间比较”；
- 对小样本、空分组、终止偏差给出醒目提示；
- 支持从聚合结论一键追溯到对应个体回答；
- 允许研究者标注、收藏、批注并整理为待验证假设。

### 沉淀与协作阶段

- 自动保存每次运行的配置快照、结果和时间；
- 支持不同轮次、不同问卷版本、不同样本方案的对比；
- 导出原始回答、受访者属性、状态/终止原因、汇总表和配置说明；
- 生成一份可用于研究组讨论的简要报告；
- 支持项目、草稿、运行记录和报告的清晰层级。

### 3.3 当前做到了吗，做得好吗

### 已做到且方向正确

1. **问卷和访谈双模式**：两种模式的中心区域确实采用不同表达——问卷强调批量汇总与单份问卷，访谈强调受访者列表与对话详情，符合两类任务的基本差异。
2. **从配置到结果的可见闭环**：用户不需要跳转页面就能看到配置、运行和分析。
3. **运行状态反馈清楚**：顶部“运行中/待机”、按钮禁用、实时完成率和受访者状态能降低等待焦虑。
4. **个体与总体之间可切换**：问卷可以翻单份结果，访谈可以查看完整对话，这是研究审查必需能力。
5. **人口维度筛选与分组**：支持性别、年龄、职业、城市、收入筛选和分组，已经触及研究者的真实需求。
6. **表单与 JSON 双编辑方式**：简单用户可用表单，高级用户可批量编辑配置，设计方向合理。

### 部分做到，但还不够好

1. **三栏并列造成持续信息拥挤**：1280×720 视口下，问卷模式为配置、结果、分析三栏；访谈模式进一步变成配置、名单、对话、分析四个窄区域。标题、题目、筛选项和 JSON 大量截断，研究者需要在多个内部滚动区之间切换，注意力被切碎。
2. **模式切换的研究语义不清**：问卷运行结果在切到访谈模式后继续沿用，甚至保留之前的问题筛选和分组状态。用户不确定这是“同一批数据的另一种视图”，还是“启动了另一种研究方法”。模式概念与运行记录概念混在一起。
3. **分析指标仍带有访谈语义**：在问卷模式中，右侧仍显示“总对话”“访谈完成率”等词，削弱了模式一致性。
4. **筛选后的主结果不够明确**：应用筛选后，原有全局回答分布仍在，另加“筛选洞察”。这虽然保留了上下文，但没有明确突出“当前筛选结果”，容易把全局数当成筛选数。
5. **人群分析表达薄弱**：目前更多是按单一维度列出人数和若干数值，缺少样本结构、占比、完成/终止率、交叉分布和覆盖缺口。
6. **JSON 编辑缺少安全感**：进入 JSON 后可直接编辑整段配置，但界面没有明显的格式校验状态、字段说明、应用/撤销、错误定位或版本差异。
7. **保存和导出反馈不足**：动作入口存在，但用户不能明确知道保存到了哪里、保存的是哪一版、导出包含哪些数据、动作是否成功。
8. **终止状态缺少解释**：能看到有人终止，却不能从总览直接理解终止原因、终止在第几题、是否集中于某类人群。

### 尚未做到的关键任务

- 运行前的研究设计检查；
- 自定义受访者字段与政策相关属性；
- 试跑、暂停、取消、重试与失败样本管理；
- 自动保存、版本管理和轮次对比；
- 组间差异的明确比较视图；
- 从发现回溯到证据、再转为研究假设的工作流；
- 样本覆盖和终止偏差提示；
- 可供研究组直接使用的研究摘要/报告。

### 3.4 哪些功能花哨但实际不太需要

这里不是说这些功能完全无价值，而是相对于目标用户的核心任务，当前投入优先级偏高。

1. **实时逐条“表演”模拟过程**：实时增长数字和对话逐条出现适合演示，但批量问卷场景下，研究者更需要后台运行、预计耗时、异常和完成通知。逐条动画不应占据主界面中心。
2. **受访者姓名、emoji、人格标签过度突出**：在访谈审查时有一定帮助，但公共政策研究首先需要稳定的受访者编号、关键政策属性和样本分层。拟人化装饰不应压过研究字段。
3. **情感饼图作为核心概览**：对公共政策态度而言，“支持/反对/不确定”“接受条件”“政策成本承担”等结构通常比泛化的正负面情绪更有用。情绪可以保留为次级分析。
4. **大量同时可见的统计卡片**：平均值、中位数、众数、方差、P90/P70/P50 同时陈列，看似专业，但未必都服务当前题型和决策。应根据题型和研究问题选择性展示，并优先解释样本量与组间差异。
5. **始终展示所有工作区**：配置、名单、对话、分析同时存在带来“信息很多”的观感，却牺牲了阅读宽度和任务专注。研究者通常是在“设计”“运行”“分析”三个阶段之间切换，而不是同时操作全部区域。
6. **界面中的宣传性或实现导向文案**：例如“节省空间，直观阅读”“组合图表同样遵循……”更像组件说明，而不是研究信息，应移除或放入帮助文档。

### 3.5 最迫切但当前做得不好的功能

### 1. 受访者与样本方案配置

这是公共政策模拟的地基，但目前以固定字段和若干“配置组”呈现。迫切需要：自定义属性、属性类型、枚举值、配额、交叉配额、样本结构预览、重复/冲突检查，以及“政策利益相关方”模板。

### 2. 分析筛选与组间比较

已有筛选和 group by，是当前最有价值的功能之一，但交互需要重构：

- 顶部固定显示当前分析对象和有效样本数；
- “总体 / 当前筛选 / 分组比较”使用清晰的模式切换；
- 每个分组同时显示人数、占比、完成/终止和题目指标；
- 小样本组给出警告；
- 点击某个组可下钻到个体证据；
- 支持保存分析视图，避免每次重建筛选条件。

### 3. 运行记录、保存和导出

这是把演示产品变成研究工具的最低门槛。保存后必须出现可确认的记录名称、时间、配置版本和状态；历史记录应能打开、复制、比较和导出；导出必须让用户选择范围并给出成功/失败反馈。

### 4. 信息布局和任务聚焦

当前多栏布局在普通笔记本屏幕上过于拥挤。应以研究阶段组织导航：

1. 设计：研究目标、问卷/提纲、样本；
2. 运行：进度、异常、控制；
3. 分析：摘要、问题、人群、个体证据；
4. 记录：版本、历史、对比、导出。

运行时可用抽屉快速查看配置，分析时配置不应永久占据三分之一屏幕。

### 3.6 迫切需要但当前缺失的功能

按重要性排序：

1. **项目—研究轮次—配置版本的数据组织方式**；
2. **自动保存与可复现的运行快照**；
3. **自定义受访者属性、配额和样本覆盖预览**；
4. **问卷/访谈提纲导入，以及运行前设计检查**；
5. **小样本试跑、暂停、取消、重试和失败原因管理**；
6. **多轮运行对比和 A/B 版本对比**；
7. **总体、筛选和分组结果的规范比较视图**；
8. **从汇总结果下钻到个体回答的证据链**；
9. **研究者批注、标记异常、收藏片段和形成假设清单**；
10. **面向研究组的摘要报告与完整数据字典导出**；
11. **政策研究模板**，例如政策知晓—态度—理由—行为意向—公平感—可接受条件；
12. **方法与风险说明区**，明确 AI 模拟只能用于探索、预实验和假设生成，不能替代真实代表性调查。

## 4. 关键交互问题清单

| 问题 | 用户影响 | 严重度 |
| --- | --- | --- |
| 多栏布局导致内容截断、多个内部滚动区并存 | 难以完整阅读题目、配置、对话和分析，认知负担高 | 高 |
| 问卷模式仍使用“总对话/访谈完成率”等指标 | 模式语义混乱，降低专业感 | 中 |
| 问卷和访谈切换后沿用同一批结果与筛选状态 | 用户不清楚是在切视图还是切研究方法，易误操作 | 高 |
| 筛选后全局分布仍占主位，筛选结果作为附加区出现 | 容易误读当前分析口径 | 高 |
| “不限”性别在筛选“男”时如何计入不透明 | 样本口径难解释 | 高 |
| 终止只有数量，没有原因汇总和下钻 | 无法判断终止偏差和是否需要重跑 | 高 |
| 保存动作缺少明确成功反馈，历史弹窗未清楚显示记录 | 用户不敢依赖系统沉淀研究 | 高 |
| 导出动作缺少范围、文件名和结果反馈 | 无法确认是否获得完整研究材料 | 高 |
| JSON 编辑无明显校验、应用和撤销机制 | 高级配置容易出错且缺乏安全感 | 中 |
| 人群分析缺少占比、交叉结构和完成/终止拆分 | 难以判断样本覆盖 | 高 |
| 受访者拟人化信息强，研究编号和关键政策属性弱 | 不利于严肃研究审查与引用 | 中 |
| 统计指标密集但缺少解释和决策导向 | 用户看到数字，却不知道下一步该做什么 | 中 |

## 5. 建议的目标产品结构

不建议继续在现有单页上叠加面板。更适合目标用户的结构是：

### 5.1 项目首页

- 项目名称、研究目标和负责人；
- 草稿与最近运行；
- 研究轮次列表；
- 新建、复制、导入、对比。

### 5.2 设计工作台

- 步骤一：研究目标与方法（问卷/访谈/混合）；
- 步骤二：问题或访谈提纲；
- 步骤三：受访者属性与配额；
- 步骤四：运行设置与终止规则；
- 步骤五：设计检查和样本预览；
- 主动作：先试跑，再正式运行。

### 5.3 运行中心

- 总体进度、预计剩余时间；
- 完成、终止、失败、等待；
- 终止原因和受影响人群；
- 暂停、继续、取消、重试；
- 少量实时样例，默认不展示全部过程。

### 5.4 分析工作台

- 摘要：关键分布、关键差异、异常和样本提醒；
- 问题：逐题总体与组间比较；
- 人群：样本结构、完成率、交叉维度；
- 个体：问卷/访谈原文、搜索、标注、下钻；
- 假设：收藏发现、关联证据、记录下一轮动作。

### 5.5 记录与导出

- 运行快照与版本；
- 两轮或多轮对比；
- 数据、配置、分析视图和报告导出；
- 保存状态和操作日志。

## 6. 后续迭代方向与优先级

### P0：先让研究过程可信、可控、可复现

目标：解决“我能不能放心完成一轮研究”的问题。

1. 重构为“设计—运行—分析—记录”阶段式导航，减少常驻多栏；
2. 建立项目、运行轮次和配置版本；每次运行自动保存不可变快照；
3. 补全保存、历史、导出反馈与错误状态；
4. 明确问卷/访谈是研究方法还是结果视图，避免无提示共享状态；
5. 统一模式文案和指标；
6. 增加终止/失败原因、暂停、取消和重试；
7. 规范总体、筛选和分组分析的口径展示；所有结果始终显示有效样本数；
8. 支持自定义受访者字段和基础配额预览。

P0 完成标准：用户可以创建一轮研究、清楚确认配置、受控运行、理解异常、保存并再次打开同一结果，且能解释所有图表当前使用的样本口径。

### P1：让系统真正帮助研究设计和发现问题

目标：从“运行模拟”升级为“完成探索性研究”。

1. 加入政策研究模板与问卷/访谈提纲导入；
2. 加入运行前设计检查；
3. 加入小样本试跑与从试跑复制为正式轮次；
4. 完善交叉配额、样本覆盖缺口和终止偏差提示；
5. 增加标准化组间比较和从图表下钻到个体证据；
6. 增加个体回答搜索、筛选、标注和收藏；
7. 支持保存筛选/分组视图；
8. 生成研究摘要、异常清单与待验证假设清单。

P1 完成标准：用户完成一轮模拟后，能在系统内回答“发现了什么、证据在哪里、有什么风险、下一轮真实调研要验证什么”。

### P2：提升研究协作与效率

目标：支持研究组持续使用，而不只是单次体验。

1. 多轮对比、问卷版本对比、样本方案对比；
2. 研究者批注、分享和报告协作；
3. 可复用模板库、变量字典和配额库；
4. 批量项目管理和后台任务通知；
5. 更完整的数据与报告导出格式；
6. 针对大样本的性能与渐进式加载优化。

---

## 7. 建议暂缓的投入

在 P0 和 P1 完成前，建议暂缓：

- 增加更多装饰性图表或实时动画；
- 扩展更多受访者头像、姓名或人格标签；
- 堆叠更多未经任务验证的统计指标；
- 继续把新功能压入同一屏多栏布局；
- 生成更华丽但不可追溯到证据的自动结论。

这些投入可以提升演示效果，却不会显著提高研究者完成核心任务的能力。

---

## 8. 最终结论

当前产品已经证明了一个有吸引力的方向：研究者可以配置问卷和样本，观察 AI 受访者完成问卷或访谈，并从总体、问题、人群和个体层面查看结果。它具备可演示的完整闭环，也有若干值得保留的基础能力。

但目标用户不会因为“模拟过程很像真的”而持续使用；他们会因为“研究设计更快被检查、样本覆盖更清楚、异常更可控、结果更容易比较、每轮工作都可复现”而持续使用。

因此，下一阶段最重要的产品转向是：

> 从 AI 受访者模拟展示器，转向面向探索性研究的 AI 调研工作台。

判断后续迭代是否成功，可以只看一个标准：一位研究者完成一轮模拟后，是否能在不离开系统的情况下，清楚回答“我研究了谁、我问了什么、这轮发生了什么、哪些发现值得验证、这份工作如何被再次打开和复现”。
