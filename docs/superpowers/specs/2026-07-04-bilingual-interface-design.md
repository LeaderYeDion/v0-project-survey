# 中英文双语界面设计

## 1. 目标与范围

为现有调研模拟平台增加中文与英文两种界面语言。登录页和登录后的完整调研工作台均提供右上角语言切换，所有固定界面文案随语言即时变化。

本期只支持：

- `zh-CN`
- `en-US`

不为未来第三种语言提前设计路由或翻译管理平台。

## 2. 明确的数据边界

需要翻译：

- 页面标题和描述；
- 导航、按钮、表单标签、占位符和帮助文字；
- 状态、空状态、错误提示和确认信息；
- 图表标题、图例、提示框和坐标说明；
- ARIA 标签及其他无障碍文字；
- 登录页及登录接口错误提示；
- 前端生成的日期、数字和百分比展示格式。

保持原样：

- 用户输入的标题、描述、问题和选项；
- 后端返回的默认问卷、问题、回答、受访者资料和对话；
- 后端返回的维度名称、历史名称和导出内容；
- 前后端既有领域协议值，例如 `"不限"`。

英文界面不会猜测或二次翻译后端文本。后端返回中文就显示中文，返回英文就显示英文。

## 3. 方案选择

采用项目内置的类型安全 i18n，不增加第三方依赖。

未采用：

- `next-intl`：适合多语言 URL 和更大规模国际化，但本期只有中英文，引入路由和依赖成本过高。
- 组件内三元表达式：实现快，但大量文案会分散、难以审计，也无法在编译期保证中英文完整对应。

## 4. 语言选择与持久化

语言选择顺序：

1. 读取 `survey_locale` Cookie；
2. Cookie 不存在时读取浏览器请求的 `Accept-Language`；
3. 语言以 `zh` 开头时选择 `zh-CN`；
4. 其他语言选择 `en-US`。

用户在右上角切换后：

- 立即更新当前 React 界面；
- 写入长期 Cookie；
- 同步更新 `<html lang>`；
- 后续刷新、登录跳转和再次访问保持该选择；
- 不重建当前运行、不清空配置、不重新请求已经展示的数据。

根布局在服务端解析 Cookie 和浏览器语言，将确定的语言作为初始值传入客户端 Provider，避免英文用户先看到中文再闪烁。

## 5. 前端结构

新增模块：

- `lib/i18n/locale.ts`
  - 定义 `Locale = "zh-CN" | "en-US"`；
  - 解析 Cookie 和浏览器语言；
  - 提供 Cookie 名称、归一化和格式化辅助。
- `lib/i18n/messages.ts`
  - 存放完整中英文词典；
  - 使用 TypeScript 类型约束两套词典结构一致；
  - 动态数量、状态句子使用类型安全函数。
- `components/locale-provider.tsx`
  - 保存当前语言；
  - 提供 `useI18n()`；
  - 切换时更新 Cookie 和 `<html lang>`。
- `components/language-switcher.tsx`
  - 统一渲染 `中文 / English` 控件；
  - 在登录页和工作台右上角复用。
- `app/login/login-page-content.tsx`
  - 承接登录页固定文案；
  - 让登录页能够使用客户端语言上下文。

调整：

- `app/layout.tsx`
  - 服务端确定初始语言；
  - 包裹 `LocaleProvider`；
  - 本地化 Metadata 和 `<html lang>`。
- `app/page.tsx`
  - 使用词典替换固定文案；
  - 在页头加入语言切换。
- `app/login/page.tsx` 与 `app/login/login-form.tsx`
  - 使用统一词典；
  - 登录请求携带当前语言；
  - 登录错误使用对应语言。
- `components/survey-config-panel.tsx`
- `components/chat-simulation-panel.tsx`
- `components/bulk-survey-panel.tsx`
- `components/analytics-panel.tsx`
  - 所有固定文案、状态、图表标签和无障碍标签改为词典读取。

领域数据不进入词典。例如配置中真实存储的 `"不限"` 保持不变，英文模式只把它对应的显示说明翻译成英文。

## 6. 请求协议

所有前端请求统一携带：

```http
Accept-Language: zh-CN
```

或：

```http
Accept-Language: en-US
```

覆盖：

- 默认模板；
- 创建、轮询和取消运行；
- 保存、列表和读取历史；
- 当前运行与历史统计查询；
- JSON/CSV 导出；
- 登录请求。

`lib/survey-api.ts` 的所有公开方法显式接收 `Locale`，由共享 `request` 方法集中写入请求头。导出请求使用同一规则。显式参数使 TypeScript 能在调用点发现遗漏。

## 7. 后端语言处理

FastAPI 增加共享语言依赖：

- 业务接口必须提供 `Accept-Language`；
- 仅接受 `zh-CN` 和 `en-US`；
- 缺失或不支持时返回结构化 HTTP 400；
- 响应通过 `Content-Language` 回显本次请求语言；
- 健康检查允许缺失并默认 `zh-CN`，避免外部探针失效。

本期后端只接收、校验并透传语言语义，不改变 Mock Engine、默认模板、回答、历史、统计或导出正文。

Next.js 登录接口读取同一请求头：

- 登录服务不可用；
- 用户名或密码错误；
- 其他登录错误；

这些错误根据 `zh-CN` 或 `en-US` 返回对应文字。缺失时使用 `zh-CN`，保持兼容。

## 8. 错误处理

- 中英文词典结构不一致时，TypeScript 编译失败。
- 前端从 Cookie 或浏览器读取到不支持的语言时，中文族归一为 `zh-CN`，其他值归一为 `en-US`。
- FastAPI 收到非法语言头时返回 400，不静默选择第三种语言。
- 前端网络错误、JSON 错误和操作失败提示使用当前 UI 语言。
- 后端业务正文和错误正文不会被前端二次翻译。
- 切换语言不会覆盖当前后端返回数据。

## 9. 测试与验收

### 单元和契约测试

- 浏览器语言识别；
- Cookie 优先级和非法值回退；
- 中英文词典结构一致；
- Survey API 所有请求携带正确 `Accept-Language`；
- 登录请求携带语言头；
- FastAPI 合法、缺失和非法语言头；
- FastAPI 响应正文保持原语言；
- 登录接口中英文错误提示；
- 固定中文文案不再散落在业务组件中，领域数据和注释使用明确白名单。

### 浏览器回归

- 中文浏览器首次进入中文；
- 非中文浏览器首次进入英文；
- 登录页右上角切换；
- 登录错误随语言变化；
- 登录后语言保持；
- 工作台配置、问卷结果、访谈、分析和历史区域完整切换；
- 当前任务运行过程中切换语言不丢失状态；
- 后端返回的中文问题和回答在英文 UI 中仍保持中文；
- Network 中登录和所有 `/survey-api/*` 请求携带正确请求头。

### 全量验证

- 后端 pytest；
- 前端和部署 Node 测试；
- TypeScript 类型检查；
- Next.js 生产构建；
- 中英文浏览器端到端回归；
- Git diff 和固定中文文案扫描。

## 10. 非目标

- 第三种语言；
- `/zh`、`/en` 路由；
- 自动翻译用户或后端数据；
- 翻译管理后台；
- 后端 Mock Engine 根据语言生成不同内容；
- 翻译导出的业务数据。
