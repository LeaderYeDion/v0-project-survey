# P0-A 响应式工作区与滚动模型设计

## 目标

在不改变模拟、分析、历史记录与导出逻辑的前提下，使 Survey Agent Simulator 在 390、768、1024、1280、1440 px 宽度下无非预期横向滚动，并让用户在移动端完成配置、启动模拟、查看结果和查看分析。

## 当前问题

2026-06-27 浏览器基线复测结果：

| 视口 | 文档宽度 | 横向溢出 | 文档高度 | 纵向溢出 |
| --- | ---: | ---: | ---: | ---: |
| 390 × 844 | 881 px | 491 px | 1303 px | 459 px |
| 768 × 720 | 881 px | 113 px | 1259 px | 539 px |
| 1024 × 720 | 1024 px | 0 | 1259 px | 539 px |
| 1280 × 720 | 1280 px | 0 | 1259 px | 539 px |
| 1440 × 720 | 1440 px | 0 | 1259 px | 539 px |

根因包括：

- 页面根节点使用 `min-h-screen`，Main 又使用固定视口高度，Header 高度被重复计入；
- Main 始终是三栏横向布局，两侧栏固定为 340 px；
- 各面板虽有内部滚动，但祖先缺少完整的 `min-h-0` 与 `overflow-hidden` 约束；
- Header 在窄屏仍把品牌、模式切换和状态放在一行；
- 访谈面板始终保留 256 px 受访者列表；
- 模式选择弹窗使用固定宽度。

## 响应式架构

### 页面骨架

根节点使用 `h-dvh overflow-hidden`，由 Header 和 Main 组成纵向 flex。Header 保持自然高度并禁止收缩；Main 使用 `flex-1 min-h-0 min-w-0 overflow-hidden`。页面本身不再滚动，配置、结果与分析面板各自承担内容滚动。

### 小屏：小于 768 px

- Header 分为品牌/状态行与模式切换行；
- Header 下显示“配置 / 结果 / 分析”顶部 Tabs；
- Main 只展示当前工作区，工作区状态保存在 `app/page.tsx`；
- 用户启动模拟后自动切换到“结果”；
- 配置、结果、分析三个工作区均占满可用宽高；
- 模式选择弹窗使用 `w-[calc(100%-2rem)] max-w-[380px]`；
- 不使用底部固定导航，避免遮挡内容和处理额外安全区。

### 中屏：768–1199 px

- Main 默认只显示结果工作区；
- Header 或 Main 工具栏提供“配置”和“分析”按钮；
- 两个按钮分别打开现有 ShadCN `Sheet`，Sheet 宽度限制在视口内；
- 配置与分析在 Sheet 内保持完整面板能力；
- Sheet 关闭后继续保留配置、筛选和结果状态；
- 结果区始终拥有完整中心宽度。

### 大屏：大于等于 1200 px

- 保留配置 340 px、结果弹性宽度、分析 340 px 的三栏结构；
- 三栏容器均增加 `min-w-0 min-h-0 overflow-hidden`；
- 各面板独立滚动，不再撑高文档；
- 访谈中心区可使用完整剩余宽度。

## 组件设计

### `app/page.tsx`

扩展现有响应式 hook，使其返回 `"mobile" | "tablet" | "desktop"` 三种布局模式；新增工作区状态 `activeWorkspace: "config" | "results" | "analytics"`，以及平板配置/分析 Sheet 的开关状态。页面负责：

- 响应式骨架与断点可见性；
- 移动顶部 Tabs；
- 平板抽屉触发器；
- 在运行开始时把移动工作区切到结果；
- 复用同一组配置、结果和分析组件，不复制业务状态。

页面根据响应式 hook 只渲染当前布局结构，避免同时挂载多份带内部状态的面板。平板 Sheet 关闭时卸载面板视图，不清除由 `app/page.tsx` 持有的业务数据。首次客户端测量完成前使用桌面结构，测量后切换到真实布局；所有业务状态均位于布局判断之外。

### `components/survey-config-panel.tsx`

- 根节点补齐 `min-h-0 min-w-0 overflow-hidden`；
- 内容滚动区补齐 `min-h-0`；
- 底部运行区保持面板内固定，不随配置内容滚走；
- 双列输入在极窄宽度下降级为单列；
- 问题标题采用弹性宽度和省略保护。

### `components/chat-simulation-panel.tsx`

- 大于等于 768 px 保留受访者列表 + 对话；
- 小于 768 px 增加“受访者 / 对话”局部切换；
- 选择受访者后自动进入对话视图；
- 无受访者时仍可打开列表空状态；
- 对话长文本允许换行，不反向撑宽容器。

### `components/bulk-survey-panel.tsx` 与 `components/analytics-panel.tsx`

- 补齐 `min-h-0 min-w-0 overflow-hidden`；
- 三列统计卡在小屏改为单列或紧凑网格；
- 面板头部动作允许换行；
- 图表与长文本不得设置会撑破视口的固定最小宽度。

## 状态与数据流

响应式状态只控制视图承载位置，不改变业务数据流：

1. 配置仍由 `app/page.tsx` 持有；
2. 问卷/访谈结果与分析仍使用现有 props；
3. 切换工作区、打开/关闭 Sheet 不重置配置和结果；
4. 切换问卷/访谈模式保持现有行为；
5. `runSimulation` 开始后在小屏切到结果，运行流程本身不变。

## 异常与边界处理

- 使用 `100dvh`，降低移动浏览器地址栏变化造成的高度跳动；
- 所有工作区同时设置 `min-w-0`，防止图表、表单和长文本撑宽祖先；
- Sheet 宽度使用 `w-[min(92vw,420px)]`，在 390 px 视口内留出关闭空间；
- 标题和模式文案允许换行；非关键副标题可在最窄屏隐藏；
- 视口跨越断点时保留 `activeWorkspace`，但桌面始终展示三栏；
- 响应式导航按钮使用明确的文本、图标与 `aria-pressed` 状态。

- `documentElement.scrollWidth === innerWidth`；
- 页面根节点不产生纵向滚动；
- 移动端顶部 Tabs 可到达三个工作区；
- 平板配置与分析 Sheet 可打开、操作和关闭；
- 桌面三栏均可独立滚动；
- 390 px 下可配置问卷、启动模拟、查看结果和分析；
- 访谈移动端可从受访者列表进入对话；

- 控制台无新增 error。
