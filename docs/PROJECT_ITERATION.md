## 当前项目现状

### 产品状态

项目当前是一套 AI 调研流程的可交互前端原型，目标用户可配置问卷与受访者画像，运行 mock 访谈或问卷模拟，并查看个体回答、聚合统计和人口维度分析。它适合演示产品闭环、验证信息架构和开展探索性体验评估，尚不具备严肃研究所需的真实 AI 服务、持久化、版本管理和可复现运行基础设施。

### 已实现能力

| 模块 | 当前实现 |
| --- | --- |
| 调研模式 | 支持问卷与访谈双模式；问卷强调批量结果，访谈强调受访者列表与对话详情 |
| 响应式工作区 | 小于 768 px 使用顶部工作区 Tabs，768–1199 px 使用结果中心与配置/分析 Sheet，1200 px 以上使用可拖拽并记忆栏宽的三栏工作台 |
| 双语界面 | 支持 `zh-CN` 与 `en-US`；首次访问按语言 Cookie 优先、浏览器语言回退选择，登录页与工作台右上角可持续切换 |
| 调研配置 | 支持标题、描述、响应时间、文本/选择/量表题，以及表单与 JSON 两种编辑方式 |
| 受访者配置 | 支持性别、年龄段、职业、城市、收入与人数的固定字段配置 |

| 结果查看 | 支持问卷单份结果、访谈完整对话、问题分析、人口画像和完成/终止状态 |

| 技术栈 | Next.js 16.1.6、React 19.2.4、TypeScript、Tailwind CSS、ShadCN/Radix UI、Recharts |

## 当前能力边界

### 2026-07-03：FastAPI Mock 后端第一版迁移

- 使用项目本地 Python 3.14.6、FastAPI 和 Pydantic 建立 `backend/`。
- 将虚拟受访者、Mock 回答、终止规则、运行编排、统计、历史和导出迁至后端。
- 前端通过 `/survey-api/*` 同源路径访问 FastAPI，只维护界面输入、选择和渲染状态。
- 历史数据可跨前端刷新保留，但 FastAPI 重启后仍会丢失。
- 本轮没有接入真实模型、数据库或正式任务队列。
- 验证包括后端 pytest、前端边界测试、TypeScript、Next.js 构建，以及问卷、历史、多维分析的浏览器端到端流程。

### 2026-07-05：中英文界面与请求语言契约

- 界面只支持 `zh-CN` 和 `en-US`。首次访问优先读取 `survey_locale` Cookie，没有 Cookie 时按浏览器语言选择；登录页与工作台右上角提供持续可用的切换入口。
- 登录 API 与所有 `/survey-api/*` 请求发送当前语言的精确 `Accept-Language`。FastAPI 业务接口要求该值为受支持语言；健康接口无需请求头并默认返回 `zh-CN`。
- 部署脚本的 FastAPI 直连、认证后的 Next.js 本地代理和公网代理健康检查统一发送 `Accept-Language: zh-CN`，同时保留原认证、loopback 和失败回滚边界。
- 双语范围是界面文案、格式化显示与部分接口错误。FastAPI 不根据语言生成或翻译业务内容，现有问卷、受访者、Mock 回答和 JSON/CSV 导出在英文界面中仍可能包含中文。

以下能力尚未实现，不能在产品介绍或演示中作为现有功能承诺：

- `lib/survey-api.ts` 已接入 FastAPI，但后端仍使用 Mock Engine，没有真实 AI 模型；
- 历史记录保存在 FastAPI `MemoryRepository` 中，前端刷新后可恢复，后端重启后丢失；
- 没有“项目—研究轮次—配置版本”的数据模型，也没有不可变运行快照；
- 产品内没有多用户登录、角色权限、协作、服务端任务队列或后台运行通知；临时公网部署只有单个共享账号和 12 小时浏览器会话；
- 运行过程没有完整的暂停、继续、取消、仅重试失败样本和预计剩余时间能力；
- 受访者字段固定，尚不支持自定义政策属性、字段类型、基础配额或交叉配额；
- 没有运行前的问卷/访谈设计检查、样本覆盖检查或试跑到正式轮次的流程；
- 分析界面尚未严格区分总体、当前筛选和分组比较口径，也缺少统一的有效样本数与小样本警告；
- 保存和导出缺少稳定的命名、范围选择、成功/失败反馈及可追溯记录；
- AI 模拟结果只适合探索、预实验和假设生成，不能替代具有代表性的真实调查。
- 没有后端业务内容翻译或多语言生成能力；切换英文不会改写已经返回或导出的内容。

### 2026-07-12：访谈推断任务 Mock 支持

- 访谈模式可在运行前启用画像推断和态度判断；默认关闭，不影响既有访谈和问卷模拟。
- 推断任务保存在本次 `SurveyConfig` 快照中，支持基于预设项新增、编辑、删除和单项启停；本轮不提供全局预设管理。
- FastAPI 新增 Mock 推断服务边界，输出 `profile` / `attitude` 两类推断结果、理由、状态和证据引用；服务接口按未来真实模型接入设计，但当前不调用真实模型或网络服务。
- 右侧分析面板新增推断页，展示任务分布、完成/失败计数和结果表；中间访谈详情展示单个受访者推断 chips 与理由。
- history、JSON 导出和 CSV 导出保留推断结果；CSV 为每个任务输出 value、reason 和 status 列。
- 当前推断结果仅为 Mock 模拟，用于产品流程验证、演示和预实验，不代表真实模型判断或真实研究结论。

## 当前最迫切的迭代模块

### P0-B：研究轮次、配置快照与保存/导出可信度

这是下一轮实施目标。P0-A 已解决基础可用性问题，但历史记录仍只存在于 FastAPI `MemoryRepository`：前端刷新后可以重新打开，FastAPI 进程重启后会丢失；系统也没有项目、研究轮次、配置版本或不可变运行快照。因此，用户仍无法跨后端生命周期可靠保存、重新打开和复现一轮研究。

#### 目标数据结构

- 项目：研究目标、负责人、创建时间与最近更新时间；
- 研究轮次：模式、状态、开始/结束时间、运行设置与结果摘要；
- 配置版本：不可变的问卷、样本方案和终止规则快照；
- 运行结果：受访者、回答、状态、终止原因和分析结果；
- 导出记录：格式、范围、文件名、生成时间与结果状态。

#### 实施范围

1. 先定义项目、轮次、配置快照和结果的稳定类型与状态迁移；
2. 每次运行创建不可变配置快照，防止运行后编辑污染历史结果；
3. 用数据库仓储替换 FastAPI `MemoryRepository`，并保持前端只依赖 `lib/survey-api.ts`；
4. 保存成功后显示记录名称、时间、版本和状态；
5. 历史记录支持重新打开、复制为新轮次和导出；
6. 导出支持范围、文件名与成功/失败反馈；
7. 为存储失败、空记录、版本不兼容和导出失败提供明确错误状态。

#### 完成定义

- 前端刷新或 FastAPI 重启后，已保存的研究轮次仍可打开；
- 每轮记录能说明使用了哪一版配置、何时运行、完成/终止情况；
- 修改当前配置不会改变历史轮次的配置快照；
- 保存和导出都有可见的进行中、成功和失败反馈；
- 历史记录可复制为新轮次，不覆盖原记录；
- 持久化与导出逻辑有自动化测试，`npm run build` 通过。

### 已完成：P0-A 响应式布局与滚动模型

- `<768 px`：配置、结果/访谈、分析使用顶部 Tabs；启动模拟后自动进入结果；
- `768–1199 px`：结果工作区保持全宽，配置和分析通过左右 Sheet 打开；
- `>=1200 px`：使用可拖拽三栏工作台，默认比例为 24% / 52% / 24%，并在浏览器中记忆用户调整；
- 页面骨架使用 `h-dvh` 与 `min-h-0/min-w-0/overflow-hidden`，各面板独立滚动；
- 配置主操作固定在面板底部；
- 移动访谈支持“受访者/对话”局部切换；
- 五个目标视口的页面级横纵溢出均为 0。

## 分阶段路线图

### P0：可信、可控、可复现

目标：用户能够在常见设备上完成一轮研究，清楚确认运行配置、结果口径与异常，并再次打开同一结果。

1. 完成响应式布局与统一滚动模型；
2. 建立项目、研究轮次、配置版本和不可变运行快照；
3. 接入持久化，补齐自动保存、历史、导出和失败反馈；
4. 统一问卷/访谈的模式语义、文案和状态隔离；
5. 增加终止/失败原因、暂停、取消和重试；
6. 规范总体、筛选和分组的口径展示及有效样本数。

阶段验收：用户可以创建、受控运行、保存、重新打开并解释一轮调研；前端刷新或 FastAPI 重启不会丢失已保存记录；所有主要结果能说明当前样本口径。

### P1：研究设计与证据发现

目标：系统不仅运行模拟，还能帮助研究者发现设计问题、样本缺口和待验证假设。

1. 支持自定义受访者属性、基础/交叉配额与覆盖预览；
2. 支持政策研究模板、问卷/提纲导入和运行前设计检查；
3. 增加小样本试跑，以及从试跑复制为正式轮次；
4. 增加小样本、空分组、覆盖缺口和终止偏差提示；
5. 支持标准化组间比较与从图表下钻到个体证据；
6. 支持个体回答搜索、标注、收藏和假设清单；
7. 生成研究摘要、异常清单与下一轮建议。

阶段验收：用户能在系统内回答“发现了什么、证据在哪里、有什么风险、下一轮真实调研要验证什么”。

### P2：协作与规模化

目标：支持研究组持续使用和多项目管理。

1. 多轮运行、问卷版本和样本方案对比；
2. 研究者批注、分享、权限和报告协作；
3. 模板库、变量字典和配额库；
4. 后台任务、批量项目和完成通知；
5. 完整数据字典与更多报告格式；
6. 大样本性能、增量分析与渐进式加载。

## 统一验收原则

- **事实一致**：文档中的已实现能力必须能从代码或运行结果验证；
- **任务闭环**：验收以用户能否完成目标任务为准，不以新增组件数量为准；
- **口径明确**：数据结果必须说明范围、状态和有效样本数；
- **异常可解释**：失败、终止、空数据和小样本必须有可见反馈；
- **可复现**：影响结果的配置、版本、时间和运行设置应可追溯；
- **多视口验证**：涉及界面的迭代至少验证 390、768、1024、1280、1440 px；
- **工程验证**：每轮至少运行相关自动化检查和 `npm run build`，并记录已知警告；
- **文档同步**：每轮提交前完成本文更新，必要时同步 README。

- **下一步**：若产品需要英文业务内容，先定义内容语言、历史快照和导出一致性契约，再扩展后端数据与生成层；产品主路线继续 P0-B。

### 2026-06-28：桌面栏宽拖拽与独立登录页

- **版本/提交**：分支 `2026_06_codex`
- **目标**：让桌面三栏可按阅读需要调整宽度，并以项目内独立登录页替代浏览器原生 Basic Auth 对话框，同时保持 Quick Tunnel 部署主链路和健康检查有效。
- **完成内容**：桌面端接入已有 `react-resizable-panels`，提供两条键盘可达的拖拽分隔线、栏宽上下限和本地布局记忆；新增与模式选择入口一致的 `/login` 页面、登录接口、12 小时 HMAC 签名 Cookie、安全站内跳转和通用错误反馈；Proxy 对 HTML 页面改为登录重定向，未认证的后台请求只返回无 Basic challenge 的 `401`，避免登录页触发浏览器原生认证框，同时继续支持部署脚本主动携带 Basic Authorization 检查；移动 Tabs、平板 Sheet 和所有调研业务状态流保持不变。
- **验证**：Node 自动化测试 23/23 通过；`npx tsc --noEmit` 和生产构建通过；Browser 验证错误/正确登录、Cookie 跳转、桌面拖拽及刷新记忆通过，配置栏在 1440 px 视口中从约 342 px 拖至 454 px；390 × 844、768 × 720、1024 × 720、1200 × 720、1440 × 900 的文档宽高均与视口一致，控制台无 error/warning。
- **遗留问题**：仍只有一个共享账号，没有多用户、审计、速率限制、找回密码或显式退出入口；`npm run lint` 因仓库未安装 ESLint 可执行文件继续失败；生产构建仍提示既有 Babel 配置警告并跳过内置类型校验，已用独立 TypeScript 检查补验。
- **下一步**：产品迭代继续 P0-B；若部署面向更多使用者，再单独设计多用户认证、退出与审计能力。

### 2026-06-28：切换为零费用 Quick Tunnel + Basic Auth

- **类型**：功能 / 安全 / 文档治理
- **版本/提交**：分支 `2026_06_codex`
- **目标**：在不购买域名、不开放公网 IP 或路由器端口的前提下，让普通公网浏览器临时访问本机应用，并由部署者维护一个共享账号密码。
- **完成内容**：用匿名 Quick Tunnel 替换 Named Tunnel、DNS、Access OTP 和邮箱白名单；新增 Next.js 16 全局 Basic Auth Proxy、随机强密码初始化、临时公网网址提取、DNS 预热、Next 16 进程身份识别、失败回滚和 Tunnel 优先关闭；重写 `deploy/` 全部实时手册。
- **验证**：自动化测试 14/14 通过；shell 语法、TypeScript 和生产构建通过；真实 Quick Tunnel 成功生成公网 HTTPS 地址，启动脚本验证本地与公网无凭据均为 401、正确凭据均为 200；启动失败回滚及停止后的 PID、网址和端口关闭验证通过。
- **遗留问题**：Quick Tunnel 每次启动更换地址，没有 SLA，最多 200 个并发中的请求且不支持 SSE；Basic Auth 只有一个共享账号，不提供用户审计、速率限制或密码找回；生产构建仍有既有 Babel 警告并跳过内置类型验证，已用独立 `tsc` 补验。
- **下一步**：日常使用按 `deploy/RUNBOOK.md` 执行；产品迭代继续 P0-B 研究轮次、不可变配置快照与持久化。
